VI(visual Interface) 编辑器

VI 编译器的三种模式

vi编辑器提供了3种模式，分别是命令模式、插入模式、底行模式，每种模式下用户所能进行的操作是不一样的。

第一次进入 vi 编辑器不要直接编辑，默认停留在命令模式。你输入的任何字母都有其特定含义，例如 dd 用来删除光标停留的行。

如果需要对文本进行编辑，则需要退出命令模式，进入编辑模式。使用命令：i 进入编辑模式。

在任何模式下，按 ESC 都是直接进入命令模式。

如果需要对文件进行：保存、退出、等操作。则需要在命令模式下输入：

  • :w 保存

以上这些指令输入完毕之后，都必须按回车确认才可以执行。

命令执行结束之后还是回到命令模式。

Git 的工作原理

Git 的代码管理方式是以版本为基本单位

  1. 把远程仓库下载到本地
  2. 从远程仓库down下来的项目
    1. 工作区
    2. 暂存区
    3. 本地仓库
  3. 默认文件的添加和修改都在工作区
  4. 如果希望进行文件备份或者是版本管理，这个时候就需要使用到暂存区和本地仓库
  5. 本地仓库中存储的是一个一个的完整的版本
  • 工作区
  • 暂存区
  • 仓库区
  • 远程仓库

默认修改的文件改动的都是工作区中的代码。

暂存区是一个临时存储的空间。

本地仓库就是真正的用来存储你的历史版本。

总结：操作 Git 的基本工作流程就是先修改文件，然后执行 git add 命令。
git add 命令会把文件加入到暂存区，接着就可以执行 git commit 命令，将文件存入文档库，
从而形成一次历史记录。

  1. 添加文件到暂存区

不能直接把工作区中的文件直接提交到本地仓库，必须先进入暂存区。

暂存区存在的意义：

  • 临时备份

  • 分批次提交

  • git add 文件名/目录

  • 也可以多次执行 git add 添加多个文件/目录 到暂存区
  • git add * 提交所有的文件到暂存区
  • git add –all 提交所有的文件到暂存区
  1. 提交暂存区中的文件形成历史记录
  • git commit -m ‘提交日志’

git commit 的作用就是把暂存区中的内容作为一个版本记录保存到本地仓库，形成历史记录。
保存完毕，暂存区中的文件被清空。

Git 记录账户信息问题

  • 默认 Git 会记录你第一次登陆的用户名密码凭据
  • 如果想要在自己的本机切换多个账户，要记得去 Windows 的 凭据管理器 中删除 Git 记录的那个账户信息
    • 然后你 push 的时候才会让你重新输入账户信息
  • 如果你不想让 Git 帮你记录账户信息，则重新安装 Git ，在安装向导的过程中，把一个选项去除 credential
